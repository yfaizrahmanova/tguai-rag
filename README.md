## О проекте

Система позволяет задавать вопросы о книгах и читательских отзывах на естественном языке и получать ответы, сформированные на основе реальных фрагментов из базы отзывов.

**Датасет:** `merged_1209.json` — 1209 размеченных читательских отзывов с аннотацией по 22 категориям (Эмоции, Сюжет, Герои, Стиль, Мораль и др.)

**Пример запросов:**
- *"Что читатели думают о главных героях?"*
- *"Как оценивается авторский стиль?"*
- *"Какие эмоции вызывают книги у читателей?"*

## Этапы реализации

### Этап 1 — Подготовка данных и база знаний
- **Загрузка и очистка** — 1209 отзывов, нормализация текста, извлечение метаданных (доминирующая метка, список меток, длина)
- **Три стратегии чанкинга:**
  - По размеру (`CharacterTextSplitter`, chunk=300)
  - По предложениям (`RecursiveCharacterTextSplitter`, chunk=200)
  - Рекурсивный (`RecursiveCharacterTextSplitter`, chunk=400) ← выбран
- **Эмбеддинги:** `paraphrase-multilingual-mpnet-base-v2` — поддерживает русский, качественные многоязычные представления
- **Векторная БД:** ChromaDB (cosine similarity, HNSW-индекс)

### Этап 2 — Система ретрива
- **Similarity Search** — поиск по косинусному сходству
- **MMR** (Maximum Marginal Relevance) — баланс релевантности и разнообразия
- **BM25** — ключевой поиск по TF-IDF
- **Гибридный поиск** — 60% векторный + 40% BM25
- **Фильтрация по метаданным** — поиск по конкретной метке (Герои, Стиль, Сюжет...)
- **Контекстное сжатие** — выбор наиболее релевантных предложений из результатов
- **Метрики:** Hit@1, Hit@3, Hit@5, MRR на 20 тестовых запросах

### Этап 3 — Интеграция с LLM
- **LLM:** OpenRouter API, модель `openai/gpt-oss-120b:free` (бесплатный tier)
- **Системный промпт:** роль литературного консультанта, правила цитирования, структура ответа
- **RAG-цепочка:** `RAGSystem` класс с поддержкой истории диалога
- **Multi-query:** агрегация результатов нескольких подзапросов
- **Обработка ошибок:** fallback при пустом ретриве, timeout, HTTP-ошибки

### Этап 4 — Анализ и оптимизация
- **LLM-as-judge** оценка: faithfulness, answer_relevancy, context_relevancy
- **Кеширование** результатов поиска (ускорение повторных запросов)
- **Батчевый поиск** для обработки нескольких запросов одновременно
- **Измерение производительности:** время поиска и время генерации

## Быстрый старт

### Требования
- Python 3.9+
- API-ключ OpenRouter (бесплатный): openrouter.ai → Keys → Create Key

### Установка

```bash
git clone https://github.com/yfaizrahmanova/tguai-rag.git
cd tguai-rag
pip install -r requirements.txt
```

### Настройка

В ноутбуке `rag_system.ipynb` найдите ячейку с конфигурацией LLM и вставьте ключ:

```python
API_KEY = "ваш_ключ_openrouter"
```

### Запуск

```bash
jupyter notebook rag_system.ipynb
```

Выполните ячейки последовательно: Этап 1 → Этап 2 → Этап 3 → Этап 4.
